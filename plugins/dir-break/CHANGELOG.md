# Changelog

## [1.1.0] - 2026-07-19

###  新增
- 组件化重构：将内联 UI 拆分为 6 个独立组件（DropArea、FolderBar、ConfigBar、PreviewPanel、ConfirmDialog、HistoryList）
- 执行确认对话框：点击解散后弹出确认窗口，避免误操作
- 操作耗时显示：成功提示中增加执行耗时
- 文件夹路径支持在资源管理器中打开
- 前后对比树：以 diff 树形式展示解散前后的文件/目录变化

###  优化
- 样式集中管理：将按钮、加载动画等全局样式移至 main.css，避免重复定义
- CSS 精简合并：减少冗余样式声明，统一使用紧凑格式

###  修复
- 修复未确认直接执行的问题，增加二次确认流程

###  新增
- 拖入文件夹或文本指令触发解散
- 单层/递归两种解散模式
- 三种文件名冲突策略：自动重命名、覆盖、跳过
- 文件预览树展示受影响文件
- 一键撤回解散操作
- 历史记录持久化存储（最近 50 条）
- 冷静蓝扁平圆角设计，支持暗色主题
- 目录也可以解散到上一层：子目录作为整体移动到父目录（单层模式），不再只提文件
- 递归模式下：文件提取到父目录后，非空子目录也会被整体移出
- 结果预览适配：目录列表标注"将移到父目录" / "将被清空"
- 操作摘要显示目录移动数量（如：移动 3 个目录）
- 结果预览全面升级：统计栏（文件数/目录数/总大小）、文件类型分布徽标
- 前后对比模块：直观展示解散前文件分布 vs 解散后结构变化
- 大量文件/目录的安全处理：限制预览数量，点击展开全部

###  修复
- 修复切换模式按钮颜色反转问题：选中为蓝色，未选中为白色（原因为 `handleToggleRecursive` 函数内部重复取反导致状态混乱）
- 修复仅含子目录时总计显示 `0 B` 的问题，改为显示 `-- 仅子目录`
- 修复 `collectMoves` 的 `isRecursing` 参数逻辑缺陷，递归模式下顶层子目录未被递归提取
- 修复递归模式执行阶段未对目录操作二次判空，导致空目录被整体搬到父目录而非删除
- **严重安全修复**：当条目名与源目录名相同时（`op.name === targetName`），`targetPath` 等于源目录。覆盖策略下 `fs.rmSync` 会误删源目录自身，现已强制转为重命名
- 修复递归扫描子目录时 `sub.dirs` 未重新计算相对于源目录的 `relativePath`，导致前端预览过滤逻辑错误
- 修复 `fs.statSync` 遇到损坏符号链接或无权限文件时崩溃，现用 try-catch 跳过并继续扫描
- 移除未使用的 `afterSummary` 计算属性、`pathDirname` 函数及 `FileInfo` 导入
